# Human Activity Recognition — MediaPipe Pose + LSTM + Streamlit

A complete, end-to-end Human Activity Recognition system that classifies human
activities from uploaded videos or a live webcam feed. Body pose is extracted
with **Google MediaPipe Pose**, and the resulting motion sequences are
classified by a **stacked LSTM** deep learning model built with
**TensorFlow/Keras**. A **Streamlit** application provides a professional,
dark-themed UI for dataset exploration, video upload, live inference, and
model analytics

Link:[https://github.com/AkkisettiGanesh/Human_Activity_Recognition](https://github.com/AkkisettiGanesh/Human_Activity_Recognition)


## Features

- 🔍 **Automatic dataset detection** — the pipeline reads whatever class
  folders and videos are present in `dataset/`; no activity names are
  hard-coded anywhere in the code.
- 🧹 **Corruption-safe preprocessing** — every video is opened and decoded
  before use; unreadable files are quarantined automatically.
- 🕺 **MediaPipe Pose landmark extraction** — 33 body keypoints per frame,
  normalized relative to the hip center, resampled to a fixed-length sequence.
- 🧠 **Stacked LSTM classifier** with dropout, L2 regularization, batch
  normalization, and EarlyStopping/ReduceLROnPlateau/ModelCheckpoint callbacks.
- 📊 **Full evaluation suite** — accuracy/loss curves, confusion matrix,
  per-class precision/recall/F1, classification report.
- 🖥️ **Streamlit app** — Home, Dataset, Upload Video, Live Webcam, Model
  Analytics, and About pages with a modern dark UI.

## Dataset

The dataset used for this project is the ZIP file provided with this task. It
follows an `<class_name>/<class_name>/*.avi` folder layout (one sub-folder per
activity, containing the video clips for that activity). The pipeline detects
this structure automatically — no class list is hard-coded.

## Project Workflow

```
dataset/ (auto-detected classes)
   -> scripts/inspect_dataset.py    (statistics + report)
   -> scripts/validate_dataset.py   (quarantine corrupted videos)
   -> scripts/split_dataset.py      (train/val/test split -> processed_data/splits.json)
   -> extract_landmarks.py          (MediaPipe Pose -> X.npy / y.npy)
   -> train.py                      (LSTM training -> models/lstm_model.keras)
   -> predict.py / app.py           (inference on video or webcam)
```

Or run every preprocessing step in one command:

```bash
python scripts/preprocess_dataset.py --dataset_dir dataset
```

## Installation

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

The MediaPipe Pose Landmarker model (`pose_landmarker_lite.task`, ~9 MB) is
downloaded automatically the first time `extract_landmarks.py` runs. If you
are offline, download it manually from the URL printed in that script's
docstring and place it at `models/pose_landmarker_lite.task`.

## Training

```bash
python scripts/preprocess_dataset.py --dataset_dir dataset   # one-time preprocessing
python train.py --epochs 100 --batch_size 32
```

This produces `models/lstm_model.keras`, `models/label_encoder.pkl`, and the
full evaluation suite in `outputs/` (accuracy/loss curves, confusion matrix,
classification report, `training_summary.json`).

## Prediction

```bash
python predict.py path/to/video.mp4
```

## Running the App

```bash
streamlit run app.py
```

## Environment Note

This repository was generated and tested against the real dataset for the
**inspection stage** (class/video counts, corruption checks, statistics —
all computed by actually running the scripts against the provided videos, see
`outputs/`). The **training stage** (MediaPipe Pose model download + full
TensorFlow LSTM training + the live Streamlit server) needs to be run in an
environment with internet access and TensorFlow/Streamlit installed — for
example your local machine or Google Colab — since it wasn't possible to
install those packages or download the MediaPipe model file in the sandbox
that generated this project. All code is complete and ready to run as-is.
